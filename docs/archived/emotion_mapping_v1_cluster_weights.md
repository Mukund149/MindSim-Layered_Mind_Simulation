# Archive: Emotion Mapping v1 — Direct Cluster Weight Mapping

## Previous Design

Emotions were generated by assigning each neuron cluster a fixed weight contribution toward each emotion. Each cluster directly mapped to multiple emotions simultaneously.

**Example:**
```
threat → { fear: 0.45, anger: 0.35, disgust: 0.10, surprise: 0.10 }
```

The emotion score was computed by multiplying each cluster's activation signal by its weight for each emotion, then summing across all clusters.

---

## Why It Was Created

This was the first working emotion generation mechanism. It was straightforward to implement and gave every cluster a deterministic emotional contribution.

---

## Problems Discovered

The system was unable to reliably detect certain emotions, particularly **sadness**. Emotions like sadness require a specific **combination** of clusters at specific activation levels — they are not just the sum of individual contributions.

The architecture also could not distinguish between "threat for fear" and "threat for anger." In both cases, high Threat activation would score fear and anger simultaneously, with no way to determine from cluster context which emotion the stimulus was actually eliciting. The system lacked sensitivity to the *pattern* of co-activated clusters, only to individual cluster magnitudes.

---

## Why It Was Replaced

The current architecture defines **emotion profiles** — each emotion is described by a specific combination of clusters, each required to be within a certain activation range. This allows the system to distinguish between situations that activate the same cluster at different levels or in different combinations, enabling more nuanced and contextually accurate emotion generation.

See [ADR-004](../adr/ADR-004-emotion-profile-based-mapping.md) for the full decision and the complete evolution timeline from v1 through to the final right-shoulder membership system.
